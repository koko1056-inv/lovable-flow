
# MGC TaskFlow - 社内タスク管理ツール

自社利用向けのタスク管理ツールを Lovable Cloud（Supabase）バックエンドで構築します。認証なしで社内共有する前提です。

## データモデル
- **members**: 担当者（名前、色、アバター）
- **projects**: プロジェクト（名前、色）
- **tags**: タグ
- **tasks**: タスク（タイトル、説明、ステータス、優先度、期日、担当者、プロジェクト、親タスクID、テンプレID参照）
- **task_tags**: タスク↔タグの中間テーブル
- **task_attachments**: ファイル添付（Supabase Storage）
- **task_comments**: コメント
- **activity_logs**: 活動ログ（変更履歴）
- **recurring_templates**: 繰り返しテンプレート（daily/weekly、生成ルール、曜日、時間）
- **generated_runs**: 自動生成の冪等性管理（テンプレID + 対象日）

## 主要機能
1. **タスクCRUD** - タイトル、説明、期日、優先度、担当者、プロジェクト、タグ
2. **子タスク（サブタスク）** - 親タスクIDで階層管理、進捗％を親に集計
3. **担当者管理** - メンバー一覧画面で追加/編集
4. **コメント & 活動ログ** - タスク詳細でやり取り、ステータスや担当者変更を自動記録
5. **ファイル添付** - Supabase Storage（パブリックバケット）にアップロード、サムネ表示
6. **タグ/プロジェクト分類** - 色分けバッジ、フィルタ条件として利用
7. **期日リマインド通知** - アプリ内通知ベル（今日/明日期限のタスク件数）+ Sonnerトースト

## 繰り返しタスク（テンプレ化＋自動生成）
- **テンプレ管理画面**で daily / weekly（曜日指定）テンプレを定義
- ページロード時 / 1時間ごとにエッジ関数 or クライアントから「今日生成すべきテンプレ」をチェックし、未生成なら tasks テーブルに新規行を作成（generated_runs で重複防止）
- 生成されたタスクは通常タスクと同様にステータス変更・コメント可能

## ビュー（4種類、ヘッダーで切替）
1. **My Tasks** - 担当者セレクタで絞り込んだ自分のタスク一覧（デフォルト画面）
2. **カンバン** - 未着手 / 進行中 / レビュー / 完了の列、ドラッグ&ドロップでステータス更新
3. **リスト/テーブル** - ソート・フィルタ（担当者、プロジェクト、タグ、期日範囲、ステータス）
4. **カレンダー** - 月表示、期日でタスクをセル配置、クリックで詳細

## UI / デザイン
- **ブランド**: 「MGC TaskFlow」ロゴをサイドバーに配置
- **レイアウト**: shadcn Sidebar（My Tasks / カンバン / リスト / カレンダー / メンバー / プロジェクト / テンプレ管理 / 設定）+ ヘッダー（検索、通知ベル、ビュー担当者切替）
- **配色**: 落ち着いたビジネス系（プライマリ：ディープブルー、アクセント：ティール）、ステータス・優先度別の色トークンを index.css のセマンティックトークンに定義
- **タスク詳細**: 右側スライドオーバー（Sheet）でインライン編集、コメント・添付・サブタスク・活動ログをタブ表示
- **モバイル対応**: サイドバー折りたたみ、カードレイアウト

## セキュリティ注意点
- 認証なしのため Supabase RLS は「全員 read/write 可能」ポリシーで開放
- 社内ネットワーク or 公開URL限定共有を前提とし、後から認証を追加できる構造に設計（タスク作成者・担当者は member_id で管理）
- ファイル添付サイズ制限・拡張子チェックをクライアント側で実装

## 実装順序
1. データベーススキーマ + Storage バケット + RLS
2. デザインシステム（カラートークン、サイドバーレイアウト）
3. メンバー / プロジェクト / タグ管理画面
4. タスクCRUD + 詳細スライドオーバー（サブタスク・コメント・添付・活動ログ）
5. 4ビュー実装（My Tasks → カンバン → リスト → カレンダー）
6. 繰り返しテンプレ管理 + 自動生成ロジック
7. 期日通知（ベル + トースト）
